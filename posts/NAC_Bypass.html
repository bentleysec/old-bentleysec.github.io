NAC Bypass

Network Access Control (NAC) provides a security layer to a network that only allows authenticated and authorized users access. NAC systems detect new devices that have been connected to the network and requires the system to be authenticated. NAC systems detect systems usually by:

  • Software clients that communicate with a NAC server
  • A DHCP proxy server that listens for DHCP requests
  • A server that listens to broadcast packets like ARP
  • An SNMP server that queries switches for new MAC addresses that show up

To bypass a NAC system an attacker will need to verify how the NAC system detects new systems. If the systems uses MAC addresses an attacker can use software to change their MAC address. Systems that use DHCP proxies could be bypassed by assigning a static IP to the attackers computer.
