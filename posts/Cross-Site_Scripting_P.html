Cross-site Scripting (XSS)

Cross-site scripting (XSS)attacks occur when web applications allow an attacker to perform HTML injection, inserting their own HTML code into a web page.

Stored/Persistent

These attacks are when the injected code is stored on the web server.

Reflected

JavaScript can be embedded into webpages simply by using the <script> tag. When the web application “reflects” this input in the form of a web page, your browser processes it as it would any other web page: it displays the text portions of the web page and executes the script portions.

DOM

The Domain Object Model (DOM) is a tool for developers to manipulate web pages as objects. Cross-Site Scripting attacks can be hidden within the DOM and then the HTML code retrieves the attack in the DOM.
